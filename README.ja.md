#### インテリジェント機能

- **スマートパス解析** — すべてのノードパラメータは UUID、パス（例: `Canvas/Panel/Button`）、または名前を受け付け、自動的に解析されます
- **UI 自動検出** — UI 親ノード配下にノードを作成する際、自動的に `cc.UITransform` を追加します
- **ビューポートコンテキスト** — ノードの作成/変更時にデザイン解像度や可視範囲を返し、範囲外の場合は自動的に警告します
- **内蔵ナレッジベース** — AI はコンポーネントプロパティ表、座標系ルール、レイアウトパターンなどの知識を照会できます
- **シーンビルダー** — JSON で完全な UI 階層を記述し、一度の呼び出しで構築、Canvas/Camera/コンポーネント設定を自動処理します
- **参照画像システム** — シーンビューに UI デザイン画像をオーバーレイし、AI がデザイン画像を見ながら画面を組み立てられます
- **ノードツリーキャッシュ** — 2 秒 TTL のキャッシュで重複クエリを回避し、変更操作後は自動的に無効化されます
- **アトミック操作** — builder/composite は snapshot 機構を使用し、失敗時は自動的にロールバックします

## 機能一覧

### 🎯 シーン操作 (scene_*)
- **scene_management**：シーン管理 - 現在のシーンの取得、シーンの開く/保存/作成/閉じる、シーン一覧のクエリに対応
- **scene_hierarchy**：シーン階層 - 完全なシーン構造の取得、コンポーネント情報の含有に対応
- **scene_execution_control**：実行制御 - コンポーネントメソッド、シーンスクリプト、プレハブ同期の実行

### 🎮 ノード操作 (node_*)
- **node_query**：ノード照会 - 名前/パターンによるノード検索、ノード情報の取得、2D/3D タイプの検出
- **node_lifecycle**：ノードライフサイクル - ノードの作成/削除、コンポーネントの事前搭載、プレハブのインスタンス化に対応
- **node_transform**：ノードトランスフォーム - ノードの名前、位置、回転、スケール、可視性などのプロパティを変更
- **node_hierarchy**：ノード階層 - ノードの移動、複製、貼り付け、階層構造の操作に対応
- **node_clipboard**：ノードクリップボード - ノードのコピー/貼り付け/切り取り操作
- **node_property_management**：プロパティ管理 - ノードプロパティ、コンポーネントプロパティ、トランスフォームプロパティのリセット

### 🔧 コンポーネント操作 (component_*)
- **component_manage**：コンポーネント管理 - エンジンコンポーネント（cc.Sprite、cc.Button など）の追加/削除
- **component_script**：スクリプトコンポーネント - カスタムスクリプトコンポーネントのアタッチ/削除
- **component_query**：コンポーネント照会 - コンポーネント一覧、詳細情報、利用可能なコンポーネントタイプの取得
- **set_component_property**：プロパティ設定 - 単一または複数のコンポーネントプロパティ値の設定

### 📦 プレハブ操作 (prefab_*)
- **prefab_browse**：プレハブ参照 - プレハブの一覧表示、情報確認、ファイル検証
- **prefab_lifecycle**：プレハブライフサイクル - ノードからのプレハブ作成、プレハブの削除
- **prefab_instance**：プレハブインスタンス - シーンへのインスタンス化、リンク解除、変更の適用、元の状態への復元
- **prefab_edit**：プレハブ編集 - 編集モードへの出入り、プレハブの保存、変更のテスト

### 🚀 プロジェクト制御 (project_*)
- **project_manage**：プロジェクト管理 - プロジェクトの実行、ビルド、プロジェクト情報・設定の取得

### 🔍 デバッグツール (debug_*)

## ⚠️ インストール前の必読事項（重要）

> **初めてインストールする場合やアップグレードする場合は、必ず現在のプロジェクトの `settings/` ディレクトリにある `mcp-server.json` と `tool-manager.json` の 2 つのファイルを削除してください。削除しないと、プラグインのツール一覧表示に異常が発生します！**
>
> ファイルパス：`你的项目/settings/mcp-server.json` と `你的项目/settings/tool-manager.json`
> これら 2 つのファイルを削除した後、プラグインパネルを開き直せば正常に戻ります。

## インストール手順

### 1. プラグインファイルのコピー

`cocos-mcp-server` フォルダ全体を、お使いの Cocos Creator プロジェクトの `extensions` ディレクトリにコピーしてください。拡張機能マネージャーから直接プロジェクトをインポートすることもできます。

```
あなたのプロジェクト/
├── assets/
├── extensions/
│   └── cocos-mcp-server/          <- ここにプラグインを配置
│       ├── source/
│       ├── dist/
│       ├── package.json
│       └── ...
├── settings/
└── ...
```

### 2. 依存関係のインストール

```bash
cd extensions/cocos-mcp-server
npm install
```

### 3. プラグインのビルド

```bash
npm run build
```

### 4. プラグインの有効化

1. Cocos Creator を再起動するか、拡張機能をリフレッシュしてください
2. プラグインが拡張機能メニューに表示されます
3. `拡張機能 > Cocos MCP Server` をクリックしてコントロールパネルを開きます

## 使用方法

### サーバーの起動

1. `拡張機能 > Cocos MCP Server` から MCP サーバーパネルを開きます
2. 設定を構成します：
   - **ポート**：HTTP サーバーポート（デフォルト：3000）
   - **自動起動**：エディタ起動時にサーバーを自動的に起動します
   - **デバッグログ**：開発デバッグのために詳細なログを有効化します
   - **最大接続数**：許可される最大同時接続数

3. 「サーバーを起動」をクリックして接続の受け付けを開始します

### AI アシスタントの接続

サーバーは `http://localhost:3000/mcp`（または設定したポート）で HTTP エンドポイントを提供します。

AI アシスタントは MCP プロトコルを使用して接続し、利用可能なすべてのツールにアクセスできます。

## トラブルシューティング

### よくある問題

1. **サーバーが起動しない**：ポートの利用可否とファイアウォール設定を確認してください
2. **ツールが動作しない**：シーンが読み込まれており、UUID が有効であることを確認してください
3. **ビルドエラー**：`npm run build` を実行して TypeScript エラーを確認してください
4. **接続の問題**：HTTP URL とサーバー状態を確認してください

### デバッグモード
