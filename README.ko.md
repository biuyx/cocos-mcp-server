## Cocos MCP 3.x Pro 버전 특징

**Cocos Creator 3.8.6+** 전용으로 제작된 전문가급 MCP(Model Context Protocol) 플러그인으로, AI 어시스턴트가 표준화된 프로토콜을 통해 에디터를 직접 조작할 수 있게 해줍니다. **16개의 의도 인식 수준 도구**가 **231개의 작업**을 커버하며 **12대 능력 모듈**을 아우르고 Cocos Creator 3.x 개발 전 과정을 관통합니다. Streamable HTTP 프로토콜을 채택하고 Token 최적화 설계를 결합하여 Token을 더 절약하고 호출이 더 안정적이며, 주요 AI 클라이언트(Cursor, Claude, Windsurf 등)의 원클릭 설정을 지원합니다.

| 의도 수준 도구 | 커버 작업 | 커버 모듈 | 통신 프로토콜 |
|:---:|:---:|:---:|:---:|
| **16** | **231** | **12대 모듈** | **Streamable HTTP** |

#### 스마트 기능

- **스마트 경로 해석** — 모든 노드 매개변수는 UUID, 경로(예: `Canvas/Panel/Button`) 또는 이름을 받아들여 자동으로 해석됩니다
- **UI 자동 감지** — UI 부모 노드 아래에 노드를 생성할 때 `cc.UITransform`을 자동으로 추가합니다
- **뷰포트 컨텍스트** — 노드를 생성/수정할 때 디자인 해상도와 가시 범위를 반환하며, 범위를 벗어나면 자동으로 경고합니다
- **내장 지식 베이스** — AI가 컴포넌트 속성표, 좌표계 규칙, 레이아웃 패턴 등의 지식을 조회할 수 있습니다
- **씬 빌더** — JSON으로 완전한 UI 계층 구조를 기술하여 한 번의 호출로 구축하며, Canvas/Camera/컴포넌트 구성을 자동으로 처리합니다
- **참조 이미지 시스템** — 씬 뷰에 UI 디자인 시안을 겹쳐 표시하여 AI가 디자인 이미지를 보면서 화면을 구성할 수 있습니다
- **노드 트리 캐시** — 2초 TTL 캐시로 중복 조회를 방지하며, 변경 작업 후 자동으로 무효화됩니다
- **원자적 작업** — builder/composite는 스냅샷 메커니즘을 사용하여 실패 시 자동으로 롤백됩니다

## 도구 체계와 작업 코드

- 모든 도구는 "카테고리_작업" 형식으로 명명되며, 매개변수는 통일된 Schema를 사용하고 여러 작업 코드(action) 전환을 지원하여 유연성과 확장성을 크게 향상시켰습니다.
- 50개의 핵심 도구가 씬, 노드, 컴포넌트, 프리팹, 에셋, 프로젝트, 디버깅, 환경설정, 서버, 메시지 브로드캐스트 등 모든 에디터 작업을 포괄합니다.
- 도구 호출 예시:

```json
{
  "tool": "node_lifecycle",
  "arguments": {
    "action": "create",
    "name": "MyNode",
    "parentUuid": "parent-uuid",
    "nodeType": "2DNode"
  }
}
```

## 주요 기능 카테고리(일부 예시)

- **scene_management**: 씬 관리(씬 가져오기/열기/저장/새로 만들기/닫기)
- **node_query / node_lifecycle / node_transform**: 노드 조회, 생성, 삭제, 속성 변경
- **component_manage / component_script / component_query**: 컴포넌트 추가/삭제, 스크립트 마운트, 컴포넌트 정보

- **asset_manage / asset_analyze**: 에셋 임포트, 삭제, 의존성 분석

## 기능 특징

### 🎯 씬 조작 (scene_*)
- **scene_management**: 씬 관리 - 현재 씬 가져오기, 씬 열기/저장/생성/닫기, 씬 목록 조회 지원
- **scene_hierarchy**: 씬 계층 구조 - 전체 씬 구조 가져오기, 컴포넌트 정보 포함 지원
- **scene_execution_control**: 실행 제어 - 컴포넌트 메서드 실행, 씬 스크립트 실행, 프리팹 동기화

### 🎮 노드 조작 (node_*)
- **node_query**: 노드 조회 - 이름/패턴으로 노드 검색, 노드 정보 가져오기, 2D/3D 타입 감지
- **node_lifecycle**: 노드 라이프사이클 - 노드 생성/삭제, 컴포넌트 사전 장착, 프리팹 인스턴스화 지원
- **node_transform**: 노드 변환 - 노드 이름, 위치, 회전, 스케일, 가시성 등의 속성 수정
- **node_hierarchy**: 노드 계층 구조 - 노드 이동, 복사, 붙여넣기, 계층 구조 조작 지원
- **node_clipboard**: 노드 클립보드 - 노드 복사/붙여넣기/잘라내기 작업
- **node_property_management**: 속성 관리 - 노드 속성, 컴포넌트 속성, 변환 속성 초기화

### 🔧 컴포넌트 조작 (component_*)
- **component_manage**: 컴포넌트 관리 - 엔진 컴포넌트 추가/삭제(cc.Sprite, cc.Button 등)
- **component_script**: 스크립트 컴포넌트 - 사용자 정의 스크립트 컴포넌트 마운트/제거
- **component_query**: 컴포넌트 조회 - 컴포넌트 목록, 상세 정보, 사용 가능한 컴포넌트 타입 가져오기
- **set_component_property**: 속성 설정 - 단일 또는 다중 컴포넌트 속성 값 설정

### 📦 프리팹 조작 (prefab_*)
- **prefab_browse**: 프리팹 브라우징 - 프리팹 목록 표시, 정보 확인, 파일 검증
- **prefab_lifecycle**: 프리팹 라이프사이클 - 노드로부터 프리팹 생성, 프리팹 삭제
- **prefab_instance**: 프리팹 인스턴스 - 씬에 인스턴스화, 링크 해제, 변경 사항 적용, 원본으로 복원
- **prefab_edit**: 프리팹 편집 - 편집 모드 진입/종료, 프리팹 저장, 변경 사항 테스트

### 🚀 프로젝트 제어 (project_*)
- **project_manage**: 프로젝트 관리 - 프로젝트 실행, 프로젝트 빌드, 프로젝트 정보 및 설정 가져오기
- **project_build_system**: 빌드 시스템 - 빌드 패널 제어, 빌드 상태 확인, 미리보기 서버 관리

### 🔍 디버그 도구 (debug_*)
- **debug_console**: 콘솔 관리 - 콘솔 로그 가져오기/지우기, 필터링 및 제한 지원
- **debug_logs**: 로그 분석 - 프로젝트 로그 파일 읽기/검색/분석, 패턴 매칭 지원
- **debug_system**: 시스템 디버그 - 에디터 정보, 성능 통계, 환경 정보 가져오기

### 📁 에셋 관리 (asset_*)
- **asset_manage**: 에셋 관리 - 에셋 일괄 임포트/삭제, 메타데이터 저장, URL 생성
- **asset_analyze**: 에셋 분석 - 의존성 관계 가져오기, 에셋 목록 내보내기
- **asset_system**: 에셋 시스템 - 에셋 새로고침, 에셋 데이터베이스 상태 조회
- **asset_query**: 에셋 조회 - 타입/폴더별 에셋 조회, 상세 정보 가져오기
- **asset_operations**: 에셋 작업 - 에셋 생성/복사/이동/삭제/저장/재임포트

### ⚙️ 환경설정 (preferences_*)
- **preferences_manage**: 환경설정 관리 - 에디터 환경설정 가져오기/설정
- **preferences_global**: 전역 설정 - 전역 구성 및 시스템 설정 관리

### 🌐 서버 및 브로드캐스트 (server_* / broadcast_*)

> **최초 설치 또는 업그레이드 전에, 반드시 현재 프로젝트의 `settings/` 디렉터리에 있는 `mcp-server.json`과 `tool-manager.json` 두 파일을 삭제하세요. 그렇지 않으면 플러그인의 도구 목록 표시에 이상이 발생합니다!**
>
> 파일 경로: `당신의 프로젝트/settings/mcp-server.json`과 `당신의 프로젝트/settings/tool-manager.json`
> 이 두 파일을 삭제한 후 플러그인 패널을 다시 열면 정상으로 복구됩니다.

## 설치 안내

### 1. 플러그인 파일 복사

`cocos-mcp-server` 폴더 전체를 Cocos Creator 프로젝트의 `extensions` 디렉터리에 복사하거나, 확장 관리자에서 직접 프로젝트를 가져올 수도 있습니다:

```
당신의 프로젝트/
├── assets/
├── extensions/
│   └── cocos-mcp-server/          <- 플러그인을 여기에 배치
│       ├── source/
│       ├── dist/
│       ├── package.json
│       └── ...
├── settings/
└── ...
```

### 2. 의존성 설치

```bash
cd extensions/cocos-mcp-server
npm install
```

### 3. 플러그인 빌드

```bash
npm run build
```

### 4. 플러그인 활성화

1. Cocos Creator를 재시작하거나 확장 프로그램을 새로고침합니다
2. 플러그인이 확장 메뉴에 나타납니다
3. `확장 > Cocos MCP Server`를 클릭하여 제어판을 엽니다

## 사용 방법

### 서버 시작

1. `확장 > Cocos MCP Server`에서 MCP 서버 패널을 엽니다
2. 설정을 구성합니다:
   - **포트**: HTTP 서버 포트(기본값: 3000)
   - **자동 시작**: 에디터 시작 시 서버 자동 시작
   - **디버그 로그**: 개발 디버깅을 위해 상세 로그 활성화
   - **최대 연결 수**: 허용되는 최대 동시 연결 수

3. "서버 시작"을 클릭하면 연결 수락이 시작됩니다

### AI 어시스턴트 연결

서버는 `http://localhost:3000/mcp`(또는 설정한 포트)에서 HTTP 엔드포인트를 제공합니다.

AI 어시스턴트는 MCP 프로토콜을 사용하여 연결하고 사용 가능한 모든 도구에 접근할 수 있습니다.

## 문제 해결

### 자주 발생하는 문제

1. **서버가 시작되지 않음**: 포트 사용 가능 여부와 방화벽 설정을 확인하세요
2. **도구가 작동하지 않음**: 씬이 로드되어 있고 UUID가 유효한지 확인하세요
3. **빌드 오류**: `npm run build`를 실행하여 TypeScript 오류를 확인하세요
4. **연결 문제**: HTTP URL과 서버 상태를 확인하세요

### 디버그 모드

플러그인 패널에서 디버그 로그를 활성화하면 상세한 작업 로그를 확인할 수 있습니다.

### 디버그 도구 사용

```json
{
  "tool": "debug_get_console_logs",
  "arguments": {"limit": 50, "filter": "error"}
}
```

```json

  "arguments": {"checkMissingAssets": true}
}
```
