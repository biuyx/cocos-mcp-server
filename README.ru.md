Профессиональный плагин MCP (Model Context Protocol), созданный специально для **Cocos Creator 3.8.6+**, позволяет AI-ассистентам напрямую управлять редактором по стандартизированному протоколу. **16 инструментов уровня намерений** охватывают **231 операцию** в **12 функциональных модулях** и сопровождают весь процесс разработки на Cocos Creator 3.x. Используется протокол Streamable HTTP в сочетании с оптимизированным расходом Token — меньше расход Token, стабильнее вызовы, поддерживается настройка одним кликом для основных AI-клиентов (Cursor, Claude, Windsurf и др.).

|:---:|:---:|:---:|:---:|
| **16** | **231** | **12 модулей** | **Streamable HTTP** |

#### Интеллектуальные возможности

- **Интеллектуальное разрешение пути** — все параметры узлов принимают UUID, путь (например, `Canvas/Panel/Button`) или имя, с автоматическим разрешением
- **Автоопределение UI** — при создании узла под UI-родителем автоматически добавляется `cc.UITransform`
- **Контекст области просмотра** — при создании/изменении узла возвращаются разрешение дизайна, видимая область, автоматическое предупреждение при выходе за границы
- **Встроенная база знаний** — AI может запрашивать таблицы свойств компонентов, правила системы координат, паттерны компоновки и другие сведения
- **Конструктор сцены** — описание полной иерархии UI в формате JSON, построение за один вызов, автоматическая обработка настройки Canvas/Camera/компонентов
- **Система референс-изображений** — наложение UI-макетов дизайна на вид сцены, AI собирает интерфейс, ориентируясь на макет
- **Кэширование дерева узлов** — кэш с TTL 2 секунды избегает повторных запросов, автоматически сбрасывается после операций изменения
- **Атомарные операции** — builder/composite используют механизм snapshot, автоматический откат при сбое

## Система инструментов и коды операций

- Все инструменты названы по схеме «категория_операция», параметры используют единую Schema, поддерживается переключение между несколькими кодами операций (action), что значительно повышает гибкость и расширяемость.
- 50 core-инструментов охватывают все операции редактора: сцены, узлы, компоненты, префабы, ассеты, проект, отладку, настройки, сервер, широковещательные сообщения и другое.
- Пример вызова инструмента:

```json
{
  "tool": "node_lifecycle",
  "arguments": {
    "action": "create",
    "name": "MyNode",
    "parentUuid": "parent-uuid",
    "nodeType": "2DNode"
  }
}
```

## Основные категории функций (некоторые примеры)

- **scene_management**: управление сценами (получение/открытие/сохранение/создание/закрытие сцены)
- **node_query / node_lifecycle / node_transform**: запрос узлов, создание, удаление, изменение свойств
- **component_manage / component_script / component_query**: добавление/удаление компонентов, подключение скриптов, информация о компонентах
- **prefab_browse / prefab_lifecycle / prefab_instance**: просмотр префабов, создание, инстанцирование, синхронизация
- **asset_manage / asset_analyze**: импорт ассетов, удаление, анализ зависимостей
- **project_manage / project_build_system**: запуск проекта, сборка, информация о конфигурации
- **debug_console / debug_logs**: управление консолью и логами
- **preferences_manage**: настройки
- **server_info**: информация о сервере
- **broadcast_message**: широковещательные сообщения


### v1.4.0 — 26 июля 2025 г.

#### 🎯 Крупные исправления функциональности
- **Полное исправление функции создания префабов**: полностью решена проблема потери ссылок на компоненты/узлы/ресурсы при создании префаба
- **Корректная обработка ссылок**: реализован формат ссылок, полностью совпадающий с форматом при ручном создании префаба
  - **Внутренние ссылки**: ссылки на узлы и компоненты внутри префаба корректно преобразуются в формат `{"__id__": x}`
  - **Внешние ссылки**: ссылки на узлы и компоненты вне префаба корректно устанавливаются в `null`
  - **Ссылки на ресурсы**: ссылки на префабы, текстуры, sprite frame и другие ресурсы полностью сохраняют формат UUID
- **Стандартизация API удаления компонентов/скриптов**: теперь при удалении компонента/скрипта необходимо передавать cid компонента (поле type), а не имя скрипта или класса. AI и пользователям следует сначала получить поле type (cid) с помощью getComponents, а затем передать его в removeComponent. Это обеспечивает 100% точное удаление всех типов компонентов и скриптов и совместимость со всеми версиями Cocos Creator.

## Быстрый старт

**Настройка Claude CLI:**

```
claude mcp add --transport http cocos-creator http://127.0.0.1:3000/mcp（используйте номер порта, который вы настроили сами）
```

**Настройка клиента Claude:**

```
{

  "mcpServers": {

		"cocos-creator": {

 		"type": "http",

		"url": "http://127.0.0.1:3000/mcp"

		 }

	  }

}
```

**Настройка MCP для Cursor или редакторов на базе VS**

```
{

  "mcpServers": { 

   "cocos-creator": {
      "url": "http://localhost:3000/mcp"
   }
  }

}
```

## Функциональные возможности

### 🎯 Операции со сценами (scene_*)
- **scene_management**: управление сценами — получение текущей сцены, открытие/сохранение/создание/закрытие сцены, поддержка запроса списка сцен
- **scene_hierarchy**: иерархия сцены — получение полной структуры сцены, поддержка включения информации о компонентах
- **scene_execution_control**: управление выполнением — выполнение методов компонентов, скриптов сцены, синхронизация префабов

### 🎮 Операции с узлами (node_*)
- **node_query**: запрос узлов — поиск узлов по имени/шаблону, получение информации об узле, определение типа 2D/3D
- **node_lifecycle**: жизненный цикл узла — создание/удаление узла, поддержка предварительной установки компонентов, инстанцирование префабов
- **node_transform**: трансформация узла — изменение имени узла, позиции, поворота, масштаба, видимости и других свойств
- **node_hierarchy**: иерархия узлов — перемещение, копирование, вставка узлов, поддержка операций с иерархической структурой
- **node_clipboard**: буфер обмена узлов — операции копирования/вставки/вырезания узлов
- **node_property_management**: управление свойствами — сброс свойств узла, свойств компонентов, свойств трансформации

### 🔧 Операции с компонентами (component_*)
- **component_manage**: управление компонентами — добавление/удаление встроенных компонентов движка (cc.Sprite, cc.Button и др.)
- **component_script**: скриптовые компоненты — подключение/удаление пользовательских скриптовых компонентов
- **component_query**: запрос компонентов — получение списка компонентов, подробной информации, доступных типов компонентов
- **set_component_property**: настройка свойств — установка значения одного или нескольких свойств компонента

### 📦 Операции с префабами (prefab_*)
- **prefab_browse**: просмотр префабов — вывод списка префабов, просмотр информации, проверка файлов
- **prefab_lifecycle**: жизненный цикл префаба — создание префаба из узла, удаление префаба
- **prefab_instance**: экземпляр префаба — инстанцирование в сцену, отвязка от префаба, применение изменений, возврат к исходному состоянию
- **prefab_edit**: редактирование префаба — вход/выход из режима редактирования, сохранение префаба, тестирование изменений

### 🚀 Управление проектом (project_*)
- **project_manage**: управление проектом — запуск проекта, сборка проекта, получение информации о проекте и настроек
- **project_build_system**: система сборки — управление панелью сборки, проверка статуса сборки, управление сервером предпросмотра

### 🔍 Инструменты отладки (debug_*)
- **debug_console**: управление консолью — получение/очистка логов консоли, поддержка фильтрации и ограничений
- **debug_logs**: анализ логов — чтение/поиск/анализ файлов логов проекта, поддержка сопоставления по шаблону
- **debug_system**: системная отладка — получение информации о редакторе, статистики производительности, информации об окружении

### 📁 Управление ассетами (asset_*)
- **asset_manage**: управление ассетами — массовый импорт/удаление ассетов, сохранение метаданных, генерация URL
- **asset_analyze**: анализ ассетов — получение зависимостей, экспорт списка ассетов
- **asset_system**: система ассетов — обновление ассетов, запрос состояния базы данных ассетов
- **asset_query**: запрос ассетов — запрос ассетов по типу/папке, получение подробной информации
- **asset_operations**: операции с ассетами — создание/копирование/перемещение/удаление/сохранение/повторный импорт ассетов

### ⚙️ Настройки (preferences_*)
- **preferences_manage**: управление настройками — получение/установка настроек редактора
- **preferences_global**: глобальные настройки — управление глобальной конфигурацией и системными настройками

### 🌐 Сервер и широковещание (server_* / broadcast_*)
- **server_info**: информация о сервере — получение состояния сервера, деталей проекта, информации об окружении
- **broadcast_message**: широковещательные сообщения — прослушивание и рассылка пользовательских сообщений

### 🖼️ Референс-изображения (referenceImage_*)
- **reference_image_manage**: управление референс-изображениями — добавление/удаление/управление референс-изображениями в виде сцены
- **reference_image_view**: вид референс-изображений — управление отображением и редактированием референс-изображений

### 🎨 Вид сцены (sceneView_*)
- **scene_view_control**: управление видом сцены — управление инструментами Gizmo, системой координат, режимом просмотра
- **scene_view_tools**: инструменты вида сцены — управление различными инструментами и параметрами вида сцены

### ✅ Инструменты валидации (validation_*)
- **validation_scene**: валидация сцены — проверка целостности сцены, проверка отсутствующих ассетов
- **validation_asset**: валидация ассетов — проверка ссылок на ассеты, проверка целостности ассетов

### 🛠️ Управление инструментами
- **Система конфигурации инструментов**: выборочное включение/отключение инструментов, поддержка нескольких наборов конфигурации
- **Сохранение конфигурации**: автоматическое сохранение и загрузка конфигурации инструментов
- **Импорт и экспорт конфигурации**: поддержка функций импорта и экспорта конфигурации инструментов
- **Управление статусом в реальном времени**: обновление и синхронизация статуса инструментов в реальном времени

### 🚀 Ключевые преимущества
- **Унификация кодов операций**: все инструменты названы по схеме «категория_операция», единая Schema параметров
- **Высокая переиспользуемость**: 50 core-инструментов охватывают 99% функциональности редактора
- **Дружелюбность к AI**: понятные параметры, полная документация, простой вызов
- **Оптимизация производительности**: снижение расхода token на 50%, повышение успешности вызовов AI
- **Полная совместимость**: 100% соответствие официальному API Cocos Creator

## Инструкция по установке

### 1. Скопируйте файлы плагина

Скопируйте всю папку `cocos-mcp-server` в каталог `extensions` вашего проекта Cocos Creator. Также можно импортировать проект напрямую через менеджер расширений:

```
ваш-проект/
├── assets/
├── extensions/
│   └── cocos-mcp-server/          <- поместите плагин сюда
│       ├── source/
│       ├── dist/
│       ├── package.json
│       └── ...
├── settings/
└── ...
```

### 2. Установите зависимости

```bash
cd extensions/cocos-mcp-server
npm install
```

### 3. Соберите плагин

```bash
npm run build
```

### 4. Включите плагин

1. Перезапустите Cocos Creator или обновите расширения
2. Плагин появится в меню расширений
3. Нажмите `Расширения > Cocos MCP Server`, чтобы открыть панель управления

## Использование

### Запуск сервера

1. Откройте панель MCP-сервера через `Расширения > Cocos MCP Server`
2. Настройте параметры:
   - **Порт**: порт HTTP-сервера (по умолчанию: 3000)
   - **Автозапуск**: автоматический запуск сервера при старте редактора
   - **Отладочные логи**: включение подробного логирования для отладки при разработке
   - **Максимум подключений**: максимально допустимое число одновременных подключений

3. Нажмите «Запустить сервер», чтобы начать принимать подключения

### Подключение AI-ассистента

Сервер предоставляет HTTP-эндпоинт по адресу `http://localhost:3000/mcp` (или на настроенном вами порте).

AI-ассистенты могут подключаться по протоколу MCP и получать доступ ко всем доступным инструментам.

### Добавление нового инструмента

1. Создайте новый класс инструмента в `source/tools/`
2. Реализуйте интерфейс `ToolExecutor`
3. Добавьте инструмент в инициализацию `mcp-server.ts`
4. Инструмент будет автоматически доступен через протокол MCP

## Устранение неполадок

### Частые проблемы

1. **Сервер не запускается**: проверьте доступность порта и настройки брандмауэра
2. **Инструменты не работают**: убедитесь, что сцена загружена и UUID действителен
3. **Ошибки сборки**: запустите `npm run build`, чтобы проверить ошибки TypeScript
4. **Проблемы с подключением**: проверьте HTTP URL и состояние сервера

### Режим отладки

Включите отладочные логи на панели плагина, чтобы получить подробные логи операций.
